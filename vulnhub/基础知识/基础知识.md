**1./etc/passwd文件内容**
七个字段
```bash
用户名：密码：UID（用户ID）：GID（组ID）：描述性信息：主目录：默认Shell
[root@localhost ~]# vi /etc/passwd
#查看一下文件内容
root:x:0:0:root:/root:/bin/bash
bin:x:1:1:bin:/bin:/sbin/nologin
daemon:x:2:2:daemon:/sbin:/sbin/nologin
adm:x:3:4:adm:/var/adm:/sbin/nologin
```
第一个：用户名
第二个：密码
第三个：用户ID，是0-65535之间的数，0是root，1-499是系统用户（系统正常运行必须的用户，无法登录），500-65536，甚至更多的就是普通用户
第四个：GID 组ID，这里的是初始组ID  组分为初始组和附加组，初始组只能有一个，附加组可以多个
第五个：描述性信息，解释用户意义，作用不大
第六个：主目录，用户登录后有操作权限的访问目录
第七关：默认shell ，shell就是linux命令解释器，默认的是/bin/bash，所以说如果用的是bash的解释器，那么就相当于拥有该用户权限范围的所有的权限
**2./etc/shadow文件内容**
第一个：用户名
第二个：加密的密码 采用的是sha512哈希函数加密的密码
第三个：最后一次修改时间 基于1970年1月1日进行计算，多少天   date -d "1970-01-01 15775 days  可以换算时间
第四个：最小修改时间间隔 避免频繁修改密码
第五个：密码有效期
第六关：密码需要变更前警告天数
**3.suid**
ruid=euid=uid
ruid启动进程的是谁
euid进程的权限，能干啥
suid权限备份 方便切换权限
本质上系统就是通过识别进程的euid来判断这个进程具有什么权限 
SUID权限就是说文件在执行时能否使用文件属主的身份来执行这个文件，否则就是只能使用执行者自己的身份
可以这样理解，一个普通用户启动一个进程，此时进程的ruid是1000，也就是此时进程是由这个用户启动的，那么此时内核检测到进程开启，把进程的euid替换成这个程序的所有者的uid，比如root的uid也就是0，然后此时euid=0，然后此时就可以执行root的权限，进程结束后euid又恢复成1000，这就是一个suid提权的过程
且suid只对二进制可执行文件有效

```bash
chmod u+s filename   设置SUID位
chmod u-s filename   去掉SUID设置
```
**4.环境变量**
实际上就是我们常用命令的可执行文件的集合，比如ls，cat等命令，实际上都是一个个可执行文件，添加环境变目的就是让系统在执行对应命令时可以直接去这些目录里面找对应的可执行的文件
然后在DC-2靶机里面，使用

```bash
:set shell=/bin/bash 
:shell

export PATH=$PATH:/bin/
export PATH=$PATH:/usr/bin/
```
后，会调用子进程bash，它没有完整的环境变量，然后这时候在添加
环境变量就能让系统执行命令时自动去这两个目录下找对应的可执行文件(默认存放绝大多基础命令的目录)
