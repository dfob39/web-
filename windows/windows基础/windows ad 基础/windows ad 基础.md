
1. AD （ Active Directory  ） 活动目录，用于集中管理windows网络环境的资源和身份权限
2.域
ad的核心逻辑单元，把网络资源划分成独立管理的域，可以理解成一个公司的所有计算机构成一个域
3.域控制器
存储域的ad数据库，主要负责处理身份验证，权限管理等请求，每个域至少有一个域控制器
4.工作组
把一些具有相同特性的计算机分在一组，实现信息共享和分散管理，信息权限还是保留在每台计算机自身，但是域的话则是主从管理模式，通过域控制器集中管理域内账号和权限
5.组织单元
可以理解成域的下一级对象，一个企业是一个域，然后组织单元可以理解成企业的各个部门，或者是某个项目组成的人员集合，总之可以根据自己的需要进行分配，组织单元可以分配相应的组策略
6.子域
可以这样理解，一个域是一个公司，然后子域就是该公司的某个分公司，可以有独立的域控制器进行控制
7.林
一个林由多个域树组成，域树又是由多个网域组成，可以理解成一个集团，下面有多种类型的公司，然后公司下面还有分公司这样子
8.信任关系
单向可传递信任关系：
域A信任域B，那么b就可以访问a的资源，那么域b和域a就是被信任关系
双向可传递信任关系：
描述一个树中父域和子域之间的关系，也用于描述一个森林中顶层域之间的关系
a信任b，b信任c，那么a就信任c，且c信任a
9.DNS服务器的SRV记录
域内计算机在加入域或者身份认证或者访问域内核心服务时，客户端不知道服务器具体ip地址，无法直接使用对应服务器域名和端口号进行访问服务，比如访问域控制器， Exchange 服务器等需要通过DNS的SRV记录去访问
10.组策略
可以作用于不同的OU，且一个OU可以链接多个组策略
