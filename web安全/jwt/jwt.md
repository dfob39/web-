**1.json-web-token**，本质上是用来进行身份验证的标识，是token的一种标准化形式，而cookie是token传输和存储的载体，是文件，在http请求头里面是以键值对字符串的形式传输，浏览器自动读取的。
**2.jwt组成**
header：通常由两部分组成，一部分是令牌类型，jwt，另一部分是加密算法，一般头部采用base64编码变体进行编码，因为base64编码存在=/+等符号在URL或者http头里面会被解析成分隔符，导致传输失败。
header
```plain
{
  "alg": "HS256",
  "typ": "JWT"
}
```
pyload：载荷，一般存储一些用户信息，比如id，声明等，但不能存放重要信息比如密码等，一般也是用base64URL进行编码
pyload  载荷
```plain
{
  "sub": "666666",
  "name": "warriors",
  "admin": true
}
```
signature：签名，是对header+pyload+pyload进行hash加密，作为签名来保障安全，因为攻击者不知道密钥，所以无法篡改，这是我自己的思路（但是理论上应该也没问题的），正确的过程就是改变哈希算法，选择的是存在密钥相关的哈希算法，比如hs526算法，把前两个和.拼接起来，然后使用密钥和哈希算法进行加密，最后在用base64URL编码进行编码，然后再和前两个进行拼接成完整的jwt。
