1.约束（mysql里面只能在单独添加一列作为约束，不能加在某一列末尾、）

- **1.NOT NULL** - 指示某列不能存储 NULL 值。         列不能有空值
      在已经创建的表age中添加not null约束

```plain
ALTER TABLE Persons
MODIFY Age int NULL;
```

- **2.   UNIQUE** - 保证某列的每行必须有唯一的值。    每个表可以有多个unique约束，但只能有一个primary       key 约束
表已经创建时添加unique约束
```plain
ALTER TABLE Persons
ADD UNIQUE (P_Id)
```

- **3.PRIMARY KEY** - NOT NULL 和 UNIQUE 的结合。  确保某列（或两个列多个列的结合）有唯一标识，有   助于更容易更快速地找到表中的一个特定的记录。      每个表都应该有一个主键，并且每个表只能有一个主键    一个主键可以有多个列构成
在 "P_Id" 列创建 PRIMARY KEY 约束
```plain
ALTER TABLE Persons
ADD PRIMARY KEY (P_Id)
```
撤销约束： ALTER TABLE Persons                   DROP PRIMARY KEY  
撤销使用drop关键字

- **.FOREIGN KEY（外键）**      - 保证一个表中的数据匹配另一个表中的值的参照完整 性。

- **5.CHECK** - 保证列中的值符合指定的条件。        CHECK 约束用于限制列中的值的范围  
在 "P_Id" 列创建 CHECK 约束
```plain
ALTER TABLE Persons
ADD CHECK (P_Id>0)
```

- **6.DEFAULT** - 规定没有给列赋值时的默认值。     表已经创建时添加default约束
mysql中
```plain
ALTER TABLE Persons
ALTER City SET DEFAULT 'SANDNES'/drop default（撤销约束）
```

- **7.INDEX** - 用于快速访问数据库表中的数据。创建索引
```plain
CREATE INDEX index_name   创建唯一索引在index前面加上unique
ON table_name (column_name)
```
删除索引
```plain
DROP INDEX [IF EXISTS] index_name
ON TABLE_NAME;
IF EXISTS：是一个可选的子句，用于检查索引是否存在
、。如果存在，就执行删除操作；如果不存在，不会报错。
```
8.truncate table 语句
只删除表中的所有数据，但保留表的结构（列，约束等）
9.alter table
修改表的列的数据类型
```plain
ALTER TABLE table_name
MODIFY COLUMN column_name datatype
```
10.auto increment 字段
 每次插入新记录时，自动地创建主键字段的值  
auto increment的开始值是1，
